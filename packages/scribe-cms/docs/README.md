# Scribe documentation

Scribe is a typed, file-based CMS for multilingual MDX sites.

- **English content lives in files.** One `.mdx` file per document; the file name is the slug. Git is your editorial history.
- **Translations live in SQLite.** A single `store.sqlite` holds every locale's frontmatter + body, generated by an LLM and keyed by a hash of the English source — so only changed pages get re-translated.
- **Schemas are Zod.** Frontmatter is validated at load time, and the schema drives everything else: which fields get translated, which fields reference other documents, and the TypeScript types of your runtime client.
- **Framework-agnostic.** The runtime is plain Node — use it from Next.js, Astro, Remix, SvelteKit, or a build script. Doc examples use Next.js, but nothing in Scribe depends on it.

## How it fits together

```
content/blog/my-post.mdx          you write this (EN)
        │
        │  scribe translate
        ▼
.scribe/store.sqlite               generated translations (fr, de, …) — commit, do not gitignore
        │
        │  createScribe(config)
        ▼
scribe.blog.resolve("mon-article", "fr")
        → { document, actualLocale, shouldRedirectTo?, canonicalPath? }
```

At runtime there is no server and no network: `createScribe()` reads the files
and the SQLite store directly, caches in memory, and revalidates on file
changes in development.
