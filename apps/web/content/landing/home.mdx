title: "Typed content for multilingual MDX sites"
description: "English content lives in MDX files in your repo. Translations live in a SQLite file next to them. Zod validates both, and a typed runtime reads everything at build time — no CMS server, no network at request time."
installCommand: "pnpm add scribe-cms zod better-sqlite3"
ctaDocs: "Read the docs"
ctaGithub: "View on GitHub"
howHeading: "How it works"
steps:
  - title: "Define a schema"
    body: "Content types are Zod schemas. Mark which fields get translated, which stay English-only, and how documents reference each other."
  - title: "Write content"
    body: "One MDX file per document, committed to your repo. The file name is the English slug; frontmatter is validated against the schema."
  - title: "Translate what changed"
    body: "Translatable content is hashed, so only missing or stale pages are sent to Gemini, through the Batch API at half the token price. Output is parsed as MDX before it is stored; a bad response fails the run, not your site. Tokens and cost are reported as it runs."
  - title: "Read it, typed"
    body: "The runtime infers document types from your config. Lists, lookups, relations, hreflang alternates, and sitemap entries all come back fully typed."
featuresHeading: "What you get"
features:
  - title: "Incremental translation"
    body: "Translations are keyed to a hash of their English source. Edit one paragraph and only that document is re-translated. Batch jobs persist in the store, so interrupted runs resume where they left off, and a dry run shows the worklist before anything is sent."
  - title: "Validated before stored"
    body: "Every translation is parsed as MDX and re-checked against your Zod schema before it lands in the store. Model mistakes fail the command, not production."
  - title: "Typed end to end"
    body: "Schemas are Zod and the runtime infers document types from the config — no codegen step. Relations between documents are dereferenced with full types."
  - title: "SEO built in"
    body: "hreflang alternates, sitemap entries with x-default, canonical paths, noindex, and redirect rules from a plain JSON file."
  - title: "Everything in git"
    body: "Content is MDX files; translations are one SQLite file you commit. English snapshots record what each translation was made from, and scribe history shows the timeline."
  - title: "No server"
    body: "Content is read from disk at build time. Works with Next.js, Astro, Remix, or any Node stack. A local read-only studio shows translation coverage and staleness."
contributeHeading: "Built in the open"
contributeBody: "scribe-cms is MIT-licensed and developed on GitHub. This site runs on it — every page here is scribe content, with a committed SQLite store behind ten locales. Read the source, open an issue, or send a pull request."
