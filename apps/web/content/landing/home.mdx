---
title: "The git-based CMS for multilingual MDX sites"
description: "English content lives in MDX files in your repo. AI translations live in a SQLite file next to them. Zod validates both, and a typed runtime reads everything at build time. No CMS server, no network at request time."
installCommand: "pnpm add scribe-cms zod better-sqlite3"
ctaDocs: "Read the docs"
ctaGithub: "View on GitHub"
howHeading: "How it works"
steps:
  - title: "Define a schema"
    body: "Content types are Zod schemas. Mark which fields get translated, which stay English-only, and how documents reference each other."
  - title: "Write content"
    body: "One MDX file per document, committed to your repo. The file name is the English slug; frontmatter is validated against the schema."
  - title: "Translate what changed"
    body: "Translatable content is hashed, so only missing or stale pages are sent to Gemini, through the Batch API at half the token price. The prompt is tuned for transcreation, so copy reads like it was written in the target language. Output is parsed as MDX before it is stored; a bad response fails the run, not your site."
  - title: "Read it, typed"
    body: "The runtime infers document types from your config. Lists, lookups, relations, hreflang alternates, and sitemap entries all come back fully typed."
featuresHeading: "What you get"
features:
  - title: "Incremental translation"
    body: "Translations are keyed to a hash of their English source. Edit one paragraph and only that document is retranslated; a dry run shows the cost first."
  - title: "Reads like a native wrote it"
    body: "The prompt is built for transcreation: a native copywriter's voice that recreates idioms instead of translating word for word."
  - title: "Validated before stored"
    body: "Every translation is parsed as MDX and re-checked against your Zod schema before it is stored. Model mistakes fail the command, not production."
  - title: "Typed end to end"
    body: "Zod schemas, inferred document types, no codegen. Relations come back dereferenced and fully typed."
  - title: "Inline tokens"
    body: "Links, assets, and untranslatable strings embedded right in prose. Edit a token and every locale updates instantly, no retranslation."
  - title: "SEO built in"
    body: "hreflang alternates, sitemaps with x-default, canonicals, noindex, JSON redirect rules, and locale fallback chains."
  - title: "Everything in git"
    body: "MDX files plus one committed SQLite store. Snapshots record what each translation was made from; scribe history shows the timeline."
  - title: "No server"
    body: "Everything is read from disk at build time; works with Next.js, Astro, or any Node stack. A local studio adds browsing, search, and coverage at a glance."
  - title: "AI-first by design"
    body: "Agents read and edit MDX with the same tools they use on code, and every change is a diff you review. No API between the model and the words."
contributeHeading: "Built in the open"
contributeBody: "scribe-cms is MIT-licensed and developed on GitHub. This site runs on it. Every page here is scribe content, with a committed SQLite store behind ten locales. Read the source, open an issue, or send a pull request."
