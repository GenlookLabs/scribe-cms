## Changed

- Rewrote the built-in translation prompt around transcreation: the model now writes as a native-speaker copywriter producing the localized edition, carrying the intent of each sentence instead of mirroring source syntax, recreating wordplay and idioms rather than rendering them literally, and applying native typographic conventions. The locale directive stays in the prompt suffix, so the shared prefix remains byte-identical across locales for prompt caching
- Prompt changes do not re-stale existing translations; run `scribe translate --force` to regenerate them with the new prompt

## Fixed

- `scribe translate --dry-run` always reported $0: dry-run items now include estimated tokens and cost based on the actual prompt and EN payload size, with the batch discount applied under `--batch`
