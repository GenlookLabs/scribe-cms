## Fixed

- `scribe translate --force` now includes fresh translations in the worklist. Previously it only skipped the per-item hash check, so a run that started from an empty worklist had nothing to retranslate

## Added

- `scribe translate --type` accepts comma-separated content type ids, e.g. `--type blog,glossary`
